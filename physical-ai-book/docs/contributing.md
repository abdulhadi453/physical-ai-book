# Contributing to Physical AI Book

We welcome contributions from the community to help improve the Physical AI Book! This guide explains how you can contribute to the project.

## Ways to Contribute

### Content Contributions
- Adding new lessons or chapters
- Improving existing content
- Adding more hands-on exercises
- Creating additional examples and case studies

### Technical Contributions
- Fixing bugs in the documentation
- Improving the website design
- Adding new components or features
- Enhancing accessibility

### Community Contributions
- Answering questions in discussions
- Reviewing pull requests
- Providing feedback on new content
- Translating content to other languages

## Getting Started

1. Fork the repository
2. Create a new branch for your contribution
3. Make your changes
4. Test your changes locally
5. Submit a pull request

## Content Guidelines

### Lesson Structure
All lessons should follow the 8-component structure:
1. Learning Objectives
2. Prerequisites
3. Theoretical Concepts
4. Real World Context
5. Hands-On Exercise
6. Exercise Solutions
7. Summary
8. Further Reading

### Writing Style
- Use clear, accessible language
- Explain concepts with examples
- Include practical, hands-on activities
- Provide multiple ways to understand concepts

## Technical Requirements

### Development Setup
```bash
# Clone your fork
git clone https://github.com/YOUR-USERNAME/physical-ai-book.git

# Navigate to the project directory
cd physical-ai-book

# Install dependencies
npm install

# Start the development server
npm start
```

### Component Usage
When creating lessons, use the custom components:
- `ExerciseBox` for hands-on exercises
- `ConceptCard` for theoretical concepts
- `ResourceLink` for further reading
- `PrerequisiteIndicator` for prerequisites
- `SummarySection` for lesson summaries

## Pull Request Process

1. Describe your changes in the pull request
2. Link any relevant issues
3. Ensure all changes follow the content guidelines
4. Wait for review and address feedback
5. Your PR will be merged once approved

## Questions?

If you have questions about contributing, feel free to open a discussion in the GitHub repository or contact the maintainers.
